# 襄行摩友小程序 - 项目规范（最终落地版）

# 一、项目核心定位与合规红线（强制遵循）

## 1.1 核心定位

- 项目定位：襄阳本地摩友公益服务工具，非盈利、纯工具属性，无交易、无社交、无导流。

- 目标用户：湖北省襄阳市本地摩托车骑行爱好者。

- 核心目标：零成本开发、100%通过微信小程序审核，打造襄阳摩友日常骑行必备工具。

## 1.2 合规红线（绝对禁止）

- 禁止交易、支付、电商、二手买卖相关功能，无任何盈利性操作。

- 禁止社交类功能（私聊、评论、点赞、关注、群聊、外部导流），不展示任何联系方式（手机号、微信、二维码）及外链。

- 禁止使用AI深度合成功能，不收集用户敏感信息（手机号、身份证号、真实姓名等）。

- 用户上传内容（照片、互助信息等）必须经管理员人工审核后展示。

- 地图仅使用腾讯地图官方SDK，不私自标注未备案点位，不做商业广告。

- 小程序简介、首页、关于页必须明确标注“非盈利公益工具”“摩友推荐非商业广告”等合规声明。

## 2.2 项目配置规范

- package.json：严格沿用现有配置，不随意升级、降级依赖，脚本命令按现有配置执行（如 dev:mp-weixin、build:mp-weixin）。

- vite.config.ts：保留现有配置（UniApp插件、别名、环境变量），优化构建速度，不额外添加冗余配置。

- pages.json：配置所有页面路由、窗口样式（默认暗黑模式）、TabBar，严格按现有页面路径配置。

- main.ts：统一引入Woto UI、uni-helper、腾讯地图SDK，初始化Bmob，不随意添加全局引入。

# 三、功能规范（按项目文档落地，禁止违规）

## 3.1 核心必做功能（第一期，已开发首页，其余按此规范）

- 首页（pages/index/index）：极简布局，顶部含襄阳定位+LOGO（融入古城墙元素），快捷入口用Woto UI卡片，公告栏滚动展示，底部含引导语及关于入口。

- 骑行地图（pages/map/map）：对接腾讯地图，标注襄阳本地骑行相关点位，实现定位、导航、筛选功能，用户上传照片需审核，底部标注合规声明。

- 骑行数据（pages/data/data）：手动/自动记录骑行数据，用图表可视化展示，支持海报生成（含襄阳元素），本地+云端备份，解锁襄阳特色成就。

- 活动工具（pages/activity/activity）：管理员发布活动，摩友匿名报名，含AA计算器、路书生成功能，无社交互动，纯工具属性。

- 口碑店铺（pages/shop/shop）：列表+地图双模式展示，仅显示地址、标签、评分，无联系方式，标注合规声明。

- 我的页面（pages/mine/mine）：展示骑行数据总览，功能入口简洁，设置功能含暗黑模式切换，视觉融入襄阳元素。

## 3.2 拓展功能（第二期，合规前提下开发）

- 物品互助角（pages/help/help）：仅支持物品赠送/互换，无价格、交易相关内容，所有信息需审核，顶部标注合规提示。

- 骑行安全工具（pages/safety/safety）：静态展示禁摩限行、应急知识、天气建议，无动态直播、复杂审核内容。

- 工具合集（pages/tools/tools）：整合油耗计算、保养提醒等高频工具，极简操作，无违规功能。

- 经典骑行线路库（pages/route/route）：分类展示襄阳特色线路，融入文化元素，支持收藏、导航，无商业广告。

## 3.3 功能开发禁忌

严格禁止开发项目文档中“绝对禁止功能”清单内的所有功能，禁止新增任何可能违反微信审核规则的功能，所有功能均需贴合“公益工具”定位。

# 四、UI设计规范（Woto UI·暗黑模式·襄阳特色）

## 4.1 设计核心原则

- 极简实用：拒绝冗余设计，操作路径最短，每个页面聚焦核心功能。

- 沉浸式暗黑：默认开启暗黑模式，所有页面、组件均适配，禁止浅色背景、刺眼元素。

- 襄阳特色：自然融入古城墙、汉江、古隆中等元素，不刻意堆砌，提升本地辨识度。

- 一致性：字体、颜色、组件样式统一，贴合Woto UI原生风格，无需自定义开发。

## 4.2 色彩规范（强制统一）

|色彩类型|色值|使用场景|
|---|---|---|
|主色调（骑行绿）|#2ED573|按钮、图标、重点文字、成就标识、导航高亮|
|辅助色（荧光橙）|#FF7A00|提醒、警告、活动标签、海报点缀|
|辅助色（襄阳灰）|#8D99AE|次要文字、边框、分割线|
|背景色（主）|#121212|页面整体背景|
|背景色（辅助）|#1E1E1E|卡片、输入框、列表项背景|
|文字色（主）|#FFFFFF|标题、正文、按钮文字|
|文字色（次）|#CCCCCC|备注、说明、次要信息|
## 4.3 字体规范

|字体类型|字号（rpx）|字重|使用场景|
|---|---|---|---|
|标题字体|32|700|页面标题、小程序名称、重点模块标题|
|副标题字体|28|600|卡片标题、功能入口标题|
|正文字体|24|400|正文、说明、备注、列表文字|
|小字体|20|400|时间、统计数字、标签文字|

## 4.4 组件与页面布局规范

- 组件规范：所有组件使用Woto UI原生组件，按钮、卡片、图标等样式统一（详见项目文档），禁止自定义基础组件。

- 页面布局：所有页面左右边距20rpx、上下边距16rpx；导航栏透明，标题居中；TabBar固定5个入口（首页、地图、数据、活动、我的），样式按色彩规范配置。

- 襄阳元素融入：LOGO、背景、成就图标、海报需融入古城墙、汉江等襄阳元素，保持自然不突兀。

- 适配规范：适配所有主流机型，使用rpx单位；加载、异常状态有友好提示，适配暗黑模式。

# 五、部署与审核规范

## 5.1 部署流程（零成本）

1. 注册微信小程序（个人主体），完成实名认证，获取AppID。

2. 在微信开发者工具中导入项目，配置AppID，调试Bmob、腾讯地图接口，确保正常运行。

3. 测试无误后，提交小程序审核（类目选择“工具-出行/旅游”）。

4. 审核通过后发布上线，无需服务器、域名、备案。

## 5.2 审核注意事项

- 合规声明：小程序简介、首页、关于页必须包含“非盈利”“非商业广告”“无交易社交”3条核心声明。

- 内容审核：上线前确认所有用户上传内容均有审核机制，无违规内容。

- 功能排查：确保无禁止功能，地图使用正规SDK，无违规导流、交易相关代码。

- 审核反馈：若审核不通过，按微信反馈修改，修改后重新提交，不添加违规内容。
