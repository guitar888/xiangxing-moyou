## 第一部分: 切换到 Bmob 真实后端 (免费方式)

### 1.1 Bmob 是什么?

Bmob (https://www.bmob.cn/) 是一个后端云服务平台,提供免费的数据存储、文件存储和云函数。免费版额度如下:

| 免费额度 | 数量 | 说明 |
|---------|------|------|
| API 请求 | 2万次/天 | 小型公益小程序绰绰有余 |
| 数据存储 | 1GB | 纯文本数据完全够用 |
| 文件存储 | 1GB | 图片等文件,建议使用外链 |
| 数据表数量 | 不限 | 免费版无限制 |
| 并发连接 | 20个/秒 | 小规模使用足够 |
| 费用 | **0元** | 完全免费 |

### 1.2 步骤 1: 注册 Bmob 并创建应用

1. 打开 https://www.bmob.cn/
2. 点击右上角「注册」,用邮箱注册账号
3. 登录后进入控制台,点击「创建应用」
4. 应用类型选择「**小程序**」
5. 应用名称填写「**襄行摩友**」

### 1.3 步骤 2: 获取密钥

1. 进入刚创建的应用
2. 左侧菜单点击「设置」→「应用密钥」
3. 记录以下两个值:
   - **Application ID** (类似 `XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX`)
   - **REST API Key** (类似 `XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX`)

### 1.4 步骤 3: 配置环境变量

项目中已预留环境变量文件,你只需要填入密钥:

**文件: `.env.development` (开发环境)**
```env
VITE_BMOB_APP_ID=你的ApplicationID
VITE_BMOB_REST_KEY=你的RESTAPIKey
```

**文件: `.env.production` (生产环境)**
```env
VITE_BMOB_APP_ID=你的ApplicationID
VITE_BMOB_REST_KEY=你的RESTAPIKey
```

### 1.6 步骤 5: 修改代码切换 Bmob

项目当前使用 Mock 数据(都在前端代码中),切换到 Bmob 需要修改 **4 个 service 文件**:

#### 修改 1: homeService.ts

将 `src/api/services/homeService.ts` 中的方法改为从 Bmob 请求真实数据:

```typescript
// 原来的 Mock 代码(删除)
// return mockActivities.filter(...)

// 改为 Bmob 请求
const response = await fetch('https://api2.bmob.cn/1/classes/Activity', {
  method: 'GET',
  headers: {
    'X-Bmob-Application-Id': import.meta.env.VITE_BMOB_APP_ID,
    'X-Bmob-REST-API-Key': import.meta.env.VITE_BMOB_REST_KEY,
    'Content-Type': 'application/json'
  }
})
const data = await response.json()
return data.results
```

每个方法都有 `// TODO: 对接 Bmob` 的注释标记,按注释提示替换即可。

#### 修改 2: mapService.ts

`src/api/services/mapService.ts` 中的路线数据同样改为从 Bmob `Route` 表读取。

#### 修改 3: rideService.ts

`src/api/services/rideService.ts` 中的骑行记录改为从 Bmob `RideRecord` 表读取。

#### 修改 4: activityService.ts

`src/api/services/activityService.ts` 中的管理员联系方式改为从 Bmob 配置表读取。

> 提示: 我已为你预留了 `// TODO: 对接 Bmob` 注释,搜索全项目找到所有需要修改的位置。

## 第二部分: 模拟数据字段详解 (供你手动修改)

当前所有数据都在代码中,以下是每个数据文件的详细字段说明。

### 2.1 活动数据

**文件位置**: `src/api/mock/activityMock.ts`
**变量名**: `mockActivities` (Array)

#### 活动字段一览

```typescript
{
  id: string,           // 【必填】唯一ID,建议 "act01" 这种格式
  title: string,        // 【必填】活动标题
  date: string,         // 【必填】日期格式 "YYYY-MM-DD"
  time: string,         // 时间,如 "19:00"
  location: string,     // 活动地点
  organizer: string,    // 组织方名称
  description: string,  // 【必填】完整描述(会显示在详情页)
  tags: string[],       // 【必填】标签数组,可选值见下方

  // -- 以下为可选字段 --
  rideType: 'city' | 'long',           // city=短途休闲, long=长途摩旅
  participantCount: number,            // 已报名人数(用于显示 "3/10人")
  maxParticipants: number,             // 最大限制人数
  gatherTime: string,                  // 集合时间描述
  gatherLocation: string,              // 集合地点
  route: string,                       // 路线描述文字
  mealInfo: string,                    // 餐饮安排
  safetyNotice: string,                // 安全提示
  disclaimer: string,                  // 免责声明
  coverImage: string,                  // 封面图URL(留空用默认图)
  routeId: string,                     // 关联的路线ID(用于跳转地图)
  routeName: string,                   // 关联的路线名称
  isFeatured: boolean,                 // 是否在首页精品推荐展示
}
```

#### tags 可选值

| 值 | 含义 | 说明 |
|----|------|------|
| `morning` | 晨骑 | 上午出发的活动 |
| `free` | 免费 | 无需报名费 |
| `gather` | 聚会 | 摩友聚会活动 |
| `scenic` | 景区 | 途经景区 |
| `greenway` | 绿道 | 沿绿道骑行 |
| `photo` | 摄影 | 摄影主题骑行 |

#### 活动状态说明

活动状态**不需要在数据中设置**,前端会根据 `date` 自动计算:
- `date` > 今天 → 即将开始 (upcoming)
- `date` = 今天 → 进行中 (ongoing)
- `date` < 今天 → 已结束 (ended)

#### 如何手动添加一个新活动

```typescript
{

  time: '08:30',                        // 时间
  location: '活动地点',
  organizer: '襄行摩友',
  description: '完整活动描述...(会显示在详情页)',
  tags: ['scenic', 'gather', 'photo'],  // 1-3个标签

  participantCount: 3,                  // 已报名人数
  maxParticipants: 10,                  // 最多限制
  gatherTime: '08:30 准时出发',
  gatherLocation: '集合地点详情',
  route: '具体路线描述',
  mealInfo: 'AA制预算约50元/人',
  safetyNotice: '佩戴头盔,遵守交规',
  disclaimer: '免责声明...',
  coverImage: '',                       // 留空=使用默认图片
  routeId: '0',                         // 关联地图路线ID
  routeName: '对应路线名称',
}
```

### 2.2 骑行路线数据

**文件位置**: `src/api/mock/mapMock.ts`
**变量名**: `mockRoutes` (Array)

#### 路线字段一览

```typescript
{
  id: string,               // 【必填】唯一ID
  name: string,             // 【必填】路线名称
  description: string,      // 【必填】完整描述(详情页展示)
  shortDescription: string, // 简短描述(卡片展示,限60字)
  region: string,           // 所属区域,可选: "gucheng"/"nanzhang"/"baokang"等
  // region 完整列表: xiangcheng,fancheng,guangcheng,yicheng,nanzhang,baokang,gucheng,laohekou,zaoyang

  distance: number,     // 总距离(公里)
  duration: number,     // 总时长(分钟)
  difficulty: 'easy' | 'medium' | 'hard',  // 难度
  loopType: 'loop' | 'oneway' | 'roundtrip', // 环线/单程/往返

  path: Array<{ latitude: number, longitude: number }>,  // 【必填】GPS坐标轨迹(用于地图绘制)

  spots: Array<{                  // 【必填】打卡点列表
    id: string,                   // 打卡点ID(唯一,如 "s0-1")
    name: string,                 // 打卡点名称
    type: 'scenic' | 'food' | 'coffee' | 'photo' | 'camp' | 'adventure',
    description: string,          // 描述
    coordinates: { latitude: number, longitude: number },  // GPS坐标
    tags?: string[]               // 标签
  }>,

  // -- 可选字段 --
  coverImage?: string,       // 封面图
  meetupPoint?: {            // 集合点
    name: string,
    address: string,
    coordinates: { latitude: number, longitude: number },
    time?: string            // 如 "09:30"
  },
  tags?: string[],           // 路线标签: mountain/coastal/forest/countryside/photo/adventure/leisure/challenge/camping/offroad
  elevation?: {              // 海拔信息
    min: number,             // 最低海拔(米)
    max: number,             // 最高海拔(米)
    gain: number             // 累计爬升(米)
  },
  roadCondition?: string,    // 路况描述
  bestSeason?: string,       // 最佳骑行季节
  highlights?: string[],     // 核心亮点列表(用于卡片展示)
  adventureAreas?: Array<{   // 探险区域
    name: string,
    type: string,
    description: string,
    coordinates: { latitude: number, longitude: number },
    difficulty?: string
  }>
}
```

## 第三部分: 上架微信小程序

### 3.1 前提准备

1. 注册微信小程序账号: https://mp.weixin.qq.com/
2. 选择「小程序」类型,填写信息时:
   - **类目**: 选择「工具」→「出行与交通」或「工具」→「信息查询」
   - **服务类目**: 不要选择需要资质的类目
3. 完成微信认证(个人主体免费,企业主体300元/年)

### 3.2 配置小程序信息

**文件: `src/manifest.json`**

需要修改以下字段:

```json
{
  // 修改应用名称
  "name": "襄行摩友",

  // 小程序配置
  "mp-weixin": {
    // 【重要】填入你的小程序 AppID(从微信公众平台获取)
    "appid": "wxXXXXXXXXXXXXXXXX",
    "setting": {
      "urlCheck": true     // 生产环境改为 true
    },
    "permission": {
      "scope.userLocation": {
        "desc": "用于骑行路线定位"
      }
    }
  }
}
```

### 3.3 构建小程序代码

```bash
# 开发模式构建
pnpm build:mp-weixin

# 生产模式构建
NODE_ENV=production pnpm build:mp-weixin
```

构建完成后,代码在 `dist/build/mp-weixin/` 目录。

### 3.4 上传到微信

1. 下载并安装「微信开发者工具」: https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 打开微信开发者工具,导入 `dist/build/mp-weixin/` 目录
3. 填入小程序的 AppID
4. 在工具中预览测试
5. 点击「上传」按钮提交代码

### 3.5 提交审核

1. 登录微信公众平台: https://mp.weixin.qq.com/
2. 进入「版本管理」
3. 选择刚才上传的版本,点击「提交审核」
4. 填写审核信息:
   - **功能描述**: "襄行摩友是襄阳地区摩友公益工具小程序,提供骑行路线查看、活动信息展示等功能"
   - **测试账号**: 如果使用 Bmob 后端无需测试账号
   - **类目确认**: 确认选择了无需资质的类目
5. 提交后等待审核(通常 1-3 个工作日)

### 3.6 审核要求与合规检查

根据项目规范,小程序必须满足以下要求:

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 无支付/交易功能 | ✅ | 无电商、无支付 |
| 无社交功能 | ✅ | 无私信、评论、点赞 |
| 无联系方式展示 | ✅ | 不展示手机号/微信号 |
| 无外链导流 | ✅ | 无外部链接跳转 |
| 有免责声明 | ✅ | 每页底部有合规声明 |
| 类目正确 | ✅ | 工具类目 |
| 隐私政策 | ✅ | 有隐私协议页面 |
